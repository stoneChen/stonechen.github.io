选择Mac/OS X的九大理由

  • mac
  • os x
OS X

使用macbook已快两年半，仅以此文为本系列开个头。

首先推荐一个权威人物：池大大，或叫卖桃君。两年多来，我一直在读他的文章，不仅仅是mac的一些知识，还有许多人文科学，受益良多。

本文将会以用户数众多的window系统作为比较，linux等系统本人不甚熟悉，酌情讨论。

正如标题所写，本文是为 OS X系统入门所撰写。对于【把mac预装的OS X系统干掉，而安装windows】的用户，你可以洗洗睡了。

  1. macbook(包括Pro,Air,或是最新的macbook)本身就是很酷的产品，科技感，便携性，不用多说。
  2. mac不需要每天关机，盖上盖子就可以起身，可谓潇洒走一回了。OS X系统具备完善的内存管理，非激活程序(即非当前焦点程序)的内存将被压缩处理，一旦被唤醒为激活程序，其内存会迅速被调出，几乎毫无痕迹。用户几乎感知不到这些切换，更大效率的使用了宝贵的内存。而windows的内存管理广为诟病，一宿不关机就得卡个半天才能反应过来。
  3. 超大的触摸板，以及灵敏的触控体验秒杀其他的notebook。最常用的就是，纵向滚动页面，无缝与手势同步。其次就是横向滚动，与纵向滚动一样轻松。无论纵向，横向还是两者同时，几乎可与手机上的滚动操作相媲美。再有，就是更加丰富高级的多手势操作，比如三指，四指，多方向滑动，pinch等操作，还附带丰富的动画效果。而windows的硬件，一般不具备如此良好的体验，带触屏的win8可能好些，未使用过，不详。
  4. OS X操作系统的稳定性。一般情况下，都不会出什么问题，我到现在为止还没重新装过系统，即使非常偶尔的死机了重新开机还是能够自动恢复至崩溃前的状态，而且系统升级是简单至极。windows的话，尤其做软件开发的，用个半年可能就要重装一次系统，太受伤了。在系统安全性方面，也比windows好上很多，根本不需要安装杀毒软件，windows稍有不慎就会中毒，杀毒软件也是防不胜防。不过客观讲，也可能跟OS X用户数少有关，黑客对用户数多的系统下手成功率要大的多。
  5. windows的软件市场太过混乱，就比如说下个迅雷，搜索到某软件中心可以下载迅雷安装包，谁知下过来的居然是一个xx软件助手，然后再通过这个助手下载真正的迅雷安装包。然后这个助手就默认开机启动了，导致系统内存浪费，忍无可忍卸载它还要花上好几分钟。这种现象太严重了，而在OS X的世界，就清净纯洁的多了。
  6. 正如卖桃君所言，OS X是GUI与命令行的完美结合体，尤其对于软件开发者来讲(windows开发者除外，据说OS X也可能运行.net环境了)。世界上最前沿的技术绝大部分是从UNIX系统率先诞生出来的（应该与它开源有关），而OS X本身就是基于UNIX的，所以命令是兼容UNIX的，那么OS X用户在享受到赏心悦目的GUI程序的同时，也能率先体验全世界开发者的软件分享。另外再说说WEB服务器，绝大多数WEB服务器是LINUX内核，通过OS X可以完美管理LINUX服务器，后期WEB后端开发者，更有理由使用OS X。从某种程度上讲，OS X就是为开发者量身定做的，首先它预装svn，git，python，ruby，Objective-C，swift自是不用说，安装java后，可开发Android和WEB服务端应用，C/C++也是不在话下。最后回到WEB前端，尤其这些年随着NodeJS的崛起，前端攻城狮也不再只是“前端”工程师了，我们可以是“全栈攻城狮”，总之OS X可谓全能开发环境。
  7. 在OS X安装软件（大部分）真的是简单的令人发指(本词也是借用卖桃君- -)。只需一个dmg，打开，把程序拖到Application目录，完毕。不过这样的方式还是少不了手动获取安装包的步骤，在OS X里可以用命令行装软件，只要啪啪敲几个单词，然后等待一小会儿，软件便自行安装完毕，具体下篇文章会讲到。
  8. 从设计行业来说，OS X都是具有其优势的。苹果公司本身就是极具设计灵魂的公司，OS X平台在很久以前就有很多优秀的设计软件，用于处理图形，音频，视频等多媒体。
  9. OS X下拥有漂亮的字体，我尤其喜欢代码的monaco字体
    简单粗暴地讲，如果你是一个设计者或软件开发者（win平台除外），使用OS X绝对不会错。而其他行业从业者，也不会不合适，或者说，OS X是一个出色的办公本。如果你想要一台游戏本，建议还是暂时回到windows的世界吧，也不是说OS X不能玩游戏，是因为国内游戏开发商基本没考虑这个平台，未来就不好说了呵呵。
