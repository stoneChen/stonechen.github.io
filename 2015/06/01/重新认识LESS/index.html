cssLESS

使用LESS已有2年，但大部分情况下使用的都是比较基础的功能，比如嵌套，变量，继承，混合，加减px/颜色值，偶尔才用函数。

今天就来重新复习一下LESS。

一、LESS是什么

LESS是一种 动态 样式语言，它的任务是将符合LESS语法的代码编译成CSS文件，简化手写传统CSS，提高样式开发的效率。

LESS 将 CSS 赋予了动态语言的特性，如 变量， 继承， 运算， 函数。LESS 既可以在 客户端 上运行 (支持IE 6+, Webkit, Firefox，动态编译LESS文件)，也可以借助Node.js或者Rhino在服务端运行（输出CSS文件，页面直接引用输出的CSS文件）。

对比一下两种方式：

客户端方式：

优点：简单

缺点：LESS编译库会读取本script标签前的所有less文件，并发起ajax读取less内容，动态地将其编译并插入到文档head中，所以当这个过程比较慢的时候（比如less内容很大，ajax响应慢等），页面会出现“无样式”的一个短暂画面，使得用户体验变得糟糕。因为在ajax发出后，文档将继续往下解析，解析文档的速度比ajax响应快的多，在ajax响应之前，被解析的文档内容都是安装浏览器原生样式显示的。

服务端方式：

优点：不会出现客户端方式的“无样式”短暂画面，用户体验好，因为是直接引用css文件。

缺点：需要额外执行编译脚本（在不引入自动化脚本的情况下，需要反复编译less文件，比较繁琐），对开发者提出了新的要求。对此，业界有一种文件监听的策略来减轻开发者的负担，即监听less文件的变动，一旦less文件有变动立即自动编译less文件生成新的css文件。比较常用的node模块有【grunt的watch插件配合grunt的less插件】，与【gulp的watch插件配合gulp的less插件】。

二、LESS的语法

LESS 做为 CSS 的一种形式的扩展，它并没有阉割 CSS 的功能，而是在现有的 CSS 语法上，添加了很多额外的功能。也就是说，LESS完全兼容CSS的语法，你在LESS里写CSS代码一点问题也没有。
