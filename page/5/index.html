使用webstorm远程开发

在前端开发过程中，有时候，我们本地开发环境无法模拟真实环境，比如没有真实的后端服务(虽然可以用nodejs模拟，但毕竟是假的)，又或者只有一部分真实环境，甚至有了全套真实环境，但数据不够真实，只能在【远程的】较真实的环境上开发，即通过不断的将本地代码应用到远程来预览效果。

比较古老的远程开发方式大致有两种：

  1. 本地代码提交到版本控制，然后通过ssh等方式登陆到远程机器，执行代码更新操作，然后浏览器刷新预览
  2. 手动通过ftp将代码传输到远程机器，然后浏览器刷新预览
    这两种方式无疑都很繁琐，不像我们本地开发时，改完代码直接切换到浏览器刷新(甚至可以实现Live Reload),就能看到效果。

接下来，该主角上场了。请注意上面的第二种方式，我使用了【手动】这个字眼。本文分享的方式，即为基于第二种方式，事先做一定配置后，即可实现近乎本地开发体验的远程开发！

一、开启远程配置

打开你的webstorm，Tools -> Deployment -> Browse Remote Host，界面如图：

二、新建远程配置

如果你还从来没使用过这个功能，左侧列表将会是空的，点击左上角的“+”，弹出一个小对话框，输入任意名字，确认。

三、配置ftp

随后在新建的远程配置中输入ftp服务所在的host地址、端口与用户名密码，并测试。

如果不成功的话，考虑切换为被动模式：

四、配置Root Path

Root Path是用于选择ftp映射的根目录，比如ftp上开启的目录是/webapps/a/b/static/…，而你本地的工程目录是从static开始，static下面的目录结构基本一样，那么你就应该把Root Path设置为/webapps/a/b/static。只有当你选择的根目录与你本地的基本一致时，远程开发才能顺利进行。

五、配置mappings

选择顶部的mappings标签

如果你希望本地工程的根目录与ftp上的某一层目录相对应即可的话（通常为同名目录），那么这里的Deployment path on serve on ‘’设置为“/”即可。

如果你希望你工程下的某一级目录跟ftp上的某一级目录对应，那么就需要设置Local path了。

点击“Use this serve as default” 按钮将这个远程配置设为默认远程主机。

六、设置自动上传

前五步已经完成了远程主机的配置，点击右下角的确定。然后菜单 -> Tools -> Deployment -> Options

将图中的下拉框选择为Always（建议）或On explicit save action（按下保存键时上传），其中Upload external changes用于配置【非手动修改时，发生改变的文件，是否自动上传】，比如less自动编译为css的时候，就非常有用。

点击确认，然后就是见证奇迹的时刻啦！观察webstorm右侧是否有Remote Host这样的一个侧边按钮，如果没有，双击shift，弹出全局搜索，输入remote，出现Remote Host选项，选择它，然后就会出现侧边栏了。

如果你上述所有的配置都正确的话，就看到类似下面这样的界面，其中绿色背景表示是被正确映射的目录/文件。

然后，随便打开一个你本地的一个文件，修改并保存，很快你就将看到，底部的状态栏，出现了：

打开浏览器， 访问你的远程主机部署的服务，检查是不是真的上传成功了。

就这样经过一番配置，我们就可以几乎像本地开发一样，远程开发，每当文件发生改变，就会立即【自动】上传到ftp服务器，换言之，即自动化了，相当便捷！

七、补充

当由于某些意外，你本地工程的代码与ftp上不一致了，我们还可以通过在左侧文件浏览器里，右键目录或文件，选择

也能够实现ftp上传（选择目录则为批量上传）。

在Remote Host栏，右键也有很多功能，请读者自行去探索，应该会给你很多惊喜。

webstorm作为前端开发神器，实至名归！

这个方法也是曾经的同事分享给我的，屡试不爽，现如今拿出来分享。

其实不仅仅前端可以使用远程开发，后端也可以，不过脚本类语言开发效率会高很多比如PHP，因为不需要编译重启等过程，JAVA就会痛苦的多，科科~

使用CSS3实现抛物线效果--自定义贝塞尔曲线

这周参与公司的一个ios Hybrid App项目，需要实现一个加入购物车时的抛物线效果。乍一看，似乎没有头绪，之前制作的CSS3动画，基本都是“直线型”的，无非在移动速率上做些效果，并没有尝试过这类“曲线”类的动画。回想起中学物理，抛物线运动可以分解为水平速度和垂直速度，水平方向是匀速的，垂直方向是加速的。于是我们只需“同时”定义这两个方向的动画即可实现抛物线。google了一番，广大同仁们的做法也是类似。给出一张分解图：

首先对我启发比较大的是 CSS3动画-抛物线运动 这篇文章（在此表示感谢）。然而，这位同仁实现的抛物线的初速度是水平的，我需要实现一个先上扬再下坠的过程，即斜上抛。于是我又陷入沉思，继续google之，发现了这个 神器 ，可以预览最终的移动轨迹，后来在张鑫旭的博客中发现了又一个神器（张鑫旭确实是一个能折腾的大神，在此也表示感谢）.这两个“神器”摆在一起，我当时却糊涂了。其实chrome开发者工具中也有类似的内置曲线图：

只不过，由起点(0,0)，终点(1,1)，活动点1(x1,y1)，活动点2(x2,y2)四个点决定这个贝塞尔曲线走向，用于描述“一个维度”的动画完成的过程，千万不要与我们这里的抛物线的轨迹混淆！我之前就是卡在了这个误区！

OS X新手必备

本文为OS X系列的第二篇，也是拖了蛮久才写。

近半年来，身边多了好几台MAC ，我都不同程度的推荐了他们应该使用的系统工具/特色、快捷键以及软件等。我们就从这两方面开个头吧，后续继续分享OS X的实用经验。

首先必须得力推卖桃君的这篇文章：开始使用Mac OS X——写给Mac新人。

然后我再写写自己一些总结。

一、系统工具/特色

  1. OS X不需要每天关机，这是我从windows迁移过来的最大变化之一。在之前的文章里我也提过，扣上盖子就可以下班回家了~得益于此，那些本身启动就很慢的大型应用，比如eclipse，idea，webstorm，photoshop等，就不必反复开启关闭了，打开盖子就可以马上干活。再举一个身边的例子，很多站点的cookie有效期，默认为至浏览器关闭前，由于我们不需要经常关机，浏览器可以长期开着，那么我们再次访问这个站点时，cookie将继续有效，不需再次登录。
  2. 合理利用spotlight。使用spotlight可以搜索日常文件与应用程序，只要你记得目标的一点蛛丝马迹，都可以通过Option + Space呼出spotlight，输入关键字找到它。不过我对它的使用也不算深入，日常使用关键字搜索足够了，更多用法请自行google。
  3. 基本可以放弃使用“桌面”了，至少我觉得没有必要了。使用windows时，我们在桌面放的最多的是应用程序的快捷方式与一些临时文件,而在OS X中，应用程序可以拖到dock（类似于win7的固定到任务栏），或通过spotlight搜索，比如打开QQ的步骤：
    Option + Space ==> QQ ==> Enter，Done！完全不需要鼠标，简单高效！
    而查找文件，仍然可以通过spotlight。通常浏览器、QQ等软件的默认下载目录都是finder侧面的“下载”目录。
  4. 合理使用OS X多桌面功能。如果你一定要使用桌面的话，OS X的多桌面功能会带给你彻底分门别类的快感，点击F3，在顶部点击+号即可添加桌面，再在新桌面上添加你需要归类的软件快捷方式。通常可用于区分不同的使用场景，比如家庭桌面，工作桌面等。
  5. 充分使用OS X自带手势操作。OS X的手势操作是相当实用与灵敏的，硕大的触摸板，给我们足够的操作空间。典型的手势是：
    双指滑动(包括垂直与水平)：鼠标滚轮
    双指轻拍：鼠标右键
    三指滑动（可配置）：点住鼠标坐标并拖动
    四指上滑：F3
    四指水平滑动：切换 桌面/全屏程序
    三指聚合：F4
    三指散开：显示桌面
  6. 暂时想到这么多。。。

二、快捷键

贴一个我早前整理的一个表格：

cmd + w

关闭子窗口/标签页

不退出程序

cmd + q

关闭整个程序


cmd + tab

切换应用程序

相当于win的alt + tab

cmd + ~

切换程序下的子窗口

比如开了多个word文档，可以在这些文档间切换，很实用

cmd + 空格

切换输入法


在finder下，cmd + shift + g

呼出跳转目录对话框


option + 空格

呼出spotlight，常用于全局搜索文件

系统自动维护索引，搜索非常快，此快捷键可更改

在spotlight搜索结果中，选择某一项，cmd + 回车/鼠标单击

打开文件/目录所在的目录





这些是比较常用的，掌握越多，效率越高~

三、必备软件

  1. homebrew：是一个命令行包管理器，通过命令行管理（安装/卸载/更新）命令行类的软件，比如通过它安装git：
    brew install git

    非软件开发人员可能觉得命令行软件不是必须的，那么接下来的homebrew cask一定是你喜欢的菜。
    官网： http://brew.sh/index_zh-cn.html
  2. homebrew cask：它是基于homebrew的命令行工具，用于安装GUI程序，比如QQ、旺旺这些带界面的软件。回首传统的软件安装方式：打开浏览器，搜索，找一个dmg文件，下载，打开，拖到Applications，一个字——繁琐！
    而通过homebrew cask（以QQ为例）：打开终端，输入brew cask install qq，回车，Done！是不是很酷？！
    官网：http://caskroom.io/
  3. iTerm2：如果你是一名软件开发者，命令行一定少不了，快捷键，搜索神马的，非常强大，以弥补系统自带的terminal的不足。
    一键安装：brew cask install iterm2
    结合zsh这个shell，如虎添翼，参考卖桃君的教程 终极 Shell

  4. office：这个不必多言，依然一键安装：

    brew cask install microsoft-office

  5. shadowsocks：用于出墙：

    brew cask install shadowsocksx

    几乎所有的出墙方式，够需要与服务端配合，shadowsocks可参考我这篇文章 DIY零代码搭建 shadowsocks服务

  6. 目前国内的网银还是个问题，有些网银仍然只支持IE，所以你需要安装虚拟机来使用网银。那么VMware Fusion是一个好选择：

    brew cask install vmware-fusion

  7. 还有输入法，笔记软件，通讯软件等等，选择很多，我就不一一罗列了，暂时讲这么多

总结

熟悉linux的童鞋，上手OS X会非常快，从windows转过来的话，可能需要花点时间，半个月足矣。

另外，我也是受卖桃君影响，觉得他说的“同是程序员，本身就应该多多互相支持”(原话可能有出入)真的有道理，所以呢，不要希望什么软件都要免费，如果你觉得一个收费软件能够大大提高你的工作效率，或是能够真正满足你的需求，就放手大胆的买下它吧。

ps：我购买过的软件/服务有：

  1. 红杏：早期出墙chrome插件
  2. 搬瓦工VPS：主要用于shadowsocks出墙
  3. 衡天虚拟空间：用于本博客
  4. 衡天域名：用于本博客
  5. webstorm：用于日常工作
  6. viscosity：用于VPN客户端
    日后将持续分享OS X的各种实用技巧、工具、软件等，欢迎交流。

angular-touch模块与hammerjs风波

前个礼拜刚开发完一个移动端项目，早前引入了angular-touch#1.2.28模块，发现点击似乎有延迟，当时没太在意，后面遇到了新的手势需求，google到hammerjs这个移动端手势神器，引入封装之（通过angular-gestures）。后来由于一系列原因又把hammer干掉了，全部采用了angular-touch模块中的ng-click和ng-swipe-left/ng-swipe-right,尚能满足需求。

与后端联调时，后端跟我反馈点击延迟严重，我便尝试升级angular#1.4.2及其所有插件(其实主要是angular-touch，但只升一个，可能会产生其他问题)，让我惊喜的是，ng-click的响应果然快了许多。当时以为忙，没细细研究原因，加入到todo list中。此次升级还遇到了onpopstate与$location服务的问题，周末研究了一下午才找到原因，并hack之，这个问题有机会另开一篇讲。

使用强大的Chrome开发者工具——杂项实用工具

本系列，这篇就来谈一谈散落在开发者工具中各个角落的实用功能。看起来似乎不起眼，但是关键时刻能够帮你解决很多问题。

以下按照我想起它们的顺序列出：

1.查找CSS选择器、属性

在Elements面板的右下角，有一个搜索框，输入关键字便可以“实时”搜索在当前子Styles这个子面板里出现的所有字符串，即使你当前的高亮节点变化了，其搜索结果也实时有效。

2.设置CSS动画速度

这也是Chrome近期的版本中新增的功能，我自己也发现没多久。还是在Elements下的Styles子面板，右上角有一个类似于播放icon的按钮。点击并选择你想要的速度(0.1~1倍速)，页面上所有的CSS动画就会相应减慢，方便你调试。

3.定位发起请求的代码

当我们需要调试框架较底层的网络请求代码，如果直接查看框架源码可能一时没有头绪。这时，我们可以通过这个功能来定位框架中的相关代码。首先在页面中做出能出触发你想要调试的网络请求，在Network面板中出现这条记录后，点击这一行的Initiator列，就会立即跳转到Sources面板的相应代码行，比如ajax一般会跳转到类似于xhr.send这样的代码行。

4.全局搜索

这个功能，可谓相当强大，可以帮我们找出一些诡异的代码来源，只要你提供一点蛛丝马迹。点击开发者工具右上角的类似命令行icon的按钮

，或当焦点在开发者工具栏上任意处时，按下ESC即可切换显示这一工具栏，选择搜索子面板。

输入搜索关键，它将在当前页面加载的所有静态资源中搜索，搜索结果将在下面列出,并高亮显示关键字，点击某一项，即可跳转到相应文件的代码行。可以看到搜索框后面，还有是否忽略大小写以及正则搜索的选项，功能强大，可见一斑！

5.查看客户端存储

开发过程中，我们可能需要用到cookie，localStorage，sessionStorage这些客户端存储。通过Chrome开发者工具可以轻松查看。一张图足以说明一切：

点击相应的存储内容，其存储内容将在右边显示。

6.代码美化

现代化的站点，上线都需要一个构建的过程，其中自然少不了代码压缩，但是这样的代码并不利于阅读。Chrome开发者工具为我们提供了美化功能，在Sources面板中打开的任意静态资源，点击左下角的一对大括号按钮，即瞬间美化
